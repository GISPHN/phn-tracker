# Googleスプレッドシート同期設定

Supabaseを使わず、GoogleスプレッドシートとApps Scriptで位置情報を共有する設定です。訓練セッションIDとアクセスコードは、アプリの「訓練作成」ボタンから自動発行できます。

## 1. Googleスプレッドシートを作る

新しいGoogleスプレッドシートを作成します。シート名は何でも構いません。

## 2. Apps Scriptを貼り付ける

スプレッドシートで `拡張機能` → `Apps Script` を開きます。

`docs/google-apps-script.gs` の内容を貼り付けて保存します。

## 3. 初期設定を実行する

Apps Script画面で関数 `setup` を選び、実行します。初回だけGoogleの承認画面が出ます。

この操作で以下の2つのシートが作成されます。

- `logs`: 位置情報とメモの記録
- `sessions`: 訓練セッションIDとアクセスコードの管理

## 4. Webアプリとしてデプロイする

`デプロイ` → `新しいデプロイ` → 種類で `ウェブアプリ` を選びます。

設定は以下にします。

- 実行するユーザー: 自分
- アクセスできるユーザー: 全員

デプロイ後に表示されるURLをコピーします。

## 5. app-config.jsにURLを入れる

```js
window.GISPHN_CONFIG = {
  mode: "gas-sheet",
  supabaseUrl: "",
  supabaseAnonKey: "",
  gasWebAppUrl: "https://script.google.com/macros/s/XXXXX/exec",
  pollIntervalMs: 30000,
};
```

## 6. 訓練を作成する

本部画面で `訓練作成` を押すと、訓練セッションIDとアクセスコードが自動発行され、`sessions` シートにも保存されます。

`共有URL` を押すと、参加者へ配布できるURLをコピーできます。

## 混在防止

スマホとPC本部画面は、同じ訓練セッションIDとアクセスコードのデータだけを共有します。

アクセスコードが違う場合は、読み取りも書き込みも拒否されます。

## 注意

この方式は訓練プロトタイプ用の簡易認証です。URLとアクセスコードを知っている人は参加できます。住民情報、詳細な健康情報、家庭訪問の実データを扱う段階では、Googleアカウント認証などの本格的なアクセス制限を追加してください。
